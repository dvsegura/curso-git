Introducción

Git,sistema de control de versiones moderno más utilizado del mundo.Git es un proyecto de código abierto maduro y con un mantenimiento activo que desarrolló originalmente Linus Torvalds, el famoso creador del kernel del sistema operativo Linux, en 2005

Presenta una arquitectura distribuida, es un ejemplo de DVCS (sistema de control de versiones distribuido, por sus siglas en inglés).

Configuración Inicial

        
          $ git --version 
          $ git config --global user.name "myName" 
          $ git config --global user.email "email"
          $ git config --global core.editor "code --wait" 
          $ git config --global -e  // consultar archivo de configuración

Configurar el core.autocrlf pues varia segun el Sistema Operativo desde donde estemos trabajando.

  • Window -> core.autocrlf true
  • Linux/Mac -> core.autocrlf input
            
                $ git config --global core.autocrlf true

Nota: Más sobre configuración con el comando

$ git config -h

Comandos básicos

  1. status or status -s(simplificado)
  2. add
  3. commit
  4. rm
  5. mv

...

git add archivo1.txt

El parametro puede ser nombre de archivo, extensión o expresión regular.
Nota: usar el "." para comprometer todos los cambios es una mala práctica. Para comprometer varios cambios a la vez, añada los nombre de archivos separados por un espacio.

Nota: cuando se hace un cambio sobre una archivo que está en Staging Area, al generarse un nuevo cambio se debe volver a actualizar el mismo.

...

$ git commit -m"Versión Inicial"
